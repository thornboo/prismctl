# 文档维护规范


本页定义 Ekko 文档的最小维护约定，目的是让贡献者与大模型能快速定位信息，并保持双语一致性。

## 目录与多语言

- 文档按语言分目录：`docs/zh-CN/` 与 `docs/en/`
- 两个目录的结构应保持镜像（同路径对应同主题）
- 共享资源放在 `docs/_shared/`（例如图片）

## 文件命名

- `docs/` 内部文件使用 `kebab-case.md`
- 每个语言目录的入口文件固定为 `README.md`
- 仓库根目录以英文为主：`README.md`、`CHANGELOG.md`、`CONTRIBUTING.md`、`LICENSE`

## 互链

- 每个页面顶部加语言互链：
  - 中文页：`> English: ...`
  - 英文页：`> 中文（简体）: ...`

## 内容结构（建议）

一页文档尽量包含：

1. 目标/适用范围
2. 快速示例（可复制）
3. 关键概念/边界（尤其是安全相关）
4. 常见问题/坑位（可选）
